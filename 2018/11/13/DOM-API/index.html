DOM API

Posted on 2018-11-13

#前言
前面有对dom的定义和概述以及dom操作进行了描述，本篇说下dom-api的几个重要知识。

1. nextSIbling和previousSibling

nextSIbling和previousSibling可能获取到的是文本节点，因为有时候标签之前还有空格，这也属于文本节点

2. innerText 和 textContent的区别

innerText 和 textContent是稍微有点区别的，前者是微软发布的，后者是火狐发布，innerText更加智能一点，会选择根据text内容选择获取，但是这样也导致它更费时间。而textContent则不会，只要是文本内容都会获取，包括script/style这种标签里的代码文本内容。

3. nodeType 主要记两个值

4. cloneNode()

参数（true）就是深拷贝，（false）/()就是浅拷贝。
浅拷贝只复制一层对象的属性，而深拷贝则递归复制了所有层级.

5. isSameNode与isEquaNode的区别

isSameNode是两个真的是一模一样（完全相同，内存地址也一样）
isEquaNode是两个看起来一模一样（内存地址不一样）

6.normalize()的作用

当前节点和它的后代节点”规范化“（normalized）。在一个”规范化”后的DOM树中，不存在一个空的文本节点，或者两个相邻的文本节点。
注1：“空的文本节点”并不包括空白字符(空格，换行等)构成的文本节点。
注2：两个以上相邻文本节点的产生原因包括：
通过脚本调用有关的DOM接口进行了文本节点的插入和分割等。
HTML中超长的文本节点会被浏览器自动分割为多个相邻文本节点。
字面翻译就是使动词|使···正常
