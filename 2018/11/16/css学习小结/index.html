css学习小结

Posted on 2018-11-16

概述

css作为前端必备技能，是很有必要去好好学习一番的，在学习的这段时间我很快就发现css其实上手并不难，但是想要学好学会真的很难，css的不正交，属性多和性价比低是主要原因。即便如此，前期还是很有必要掌握一些css基本套路的，随着工作中的时间应用css技能也会逐步提升的。

总结一些css的重要知识点或者说套路。

1.几个常见的css布局套路：两种水平居中、三种垂直居中、两种左右结构、两种左中右结构

2.控制文档流：利用好默认的宽度和高度规则

了解内联元素和块级元素以及决定他们的高度和宽度的因素

3.Z-index

堆叠上下文和堆叠顺序,只能给定位元素加z-index；

4.移动端页面（响应式）

  1. 学会媒体查询
    @meida (max-width:800px){
    body{background:red;}
    }
    @media 接受一个查询条件，如果满足条件{}里的css样式就生效；

  2. 要在手机端加一个meta标签

  3. 动态rem（手机专用的自适用方案）
    用JS把html的font-size设置等于页面宽度

  4. 了解手机端的特点例如：手机端没有hover但是有touch，手机端没后resize，没有滚动条；

    5.Flex和float布局

    flex出现之前的布局，normal flow(正常流也叫文档流)：

  5. float+clear:子元素float left 父元素加clearfix类，写好clearfix样式
  6. position定位
  7. display： inline-block 横向布局
  8. 负margin
    flxe布局
    父元素加display：flex；
    justify-content 主轴方向对齐
    align-items 侧轴方向对齐
    区别特点：
  9. 块级布局侧重垂直方向，行内布局侧重水平方向，flex布局与方向无关
  10. flex布局可以实现空间自动分配自动对齐
  11. flex布局适合简单的线性布局，更复杂的布局需要使用grid布局

    布局套路

是否要支持IE8: 是——> float布局（定宽）\
否——> flex布局（弹性宽度）

总结

css学精难，入门没那么难，日常工作应付使用的话掌握些套路足矣，后期根据需要和兴趣再另行深入学习。
